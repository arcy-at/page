7 reasons clean code matters.

What is clean code?

Clean code is a reader-focused development style that produces software that's easy to write, read and maintain. Developers are often tempted to consider their work complete when the application operates as expected. But we're not merely writing code for computer consumption. We're also writing for humans as well.

1. You're an author

An Author is someone who writes as a profession. Technically, coding / programming is writing

2. It's foundational

Clean Code pays off. Especially when it's being tested or being refactored clean code leads to software that is highly modular.

3. Justify your technique

Clean code doesn't happen on accident, it requires actual focus on what you're creating. Saying that clean code is using comments, and thats it, then you don't know what it is.

4. Reading is hard

Writing code is kinda easy. Reading it is hard. If you are unable to read the code you just wrote, how in the world are you going to let other people understand it, or even yourself after a bit?

5. Technical debt is depressing

Writing bad code gives you technical debt. In some cases, it can be good. But when you get a lot of it, it will drive the talent away. When the easiest things actually take effort to do, people start to leave.

6. You're Lazy

Sometimes, lazy can be bad. But thats the wrong kind of lazy. The good kind of lazy, the efficient kind, is applied when doing clean code. When you put extra care into the code so it doesn't become a mess. Being sloppy is never a good thing. It slows you down, By a lot.

7. Don't be a verb

Like seriously, don't. It's embarrassing. Hearing "yeah this code has been {name}d takes you feel both unconfortable and embarrassed.
